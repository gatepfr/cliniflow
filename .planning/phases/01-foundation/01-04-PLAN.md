**`packages/shared/src/errors.ts`** — operational error class:
```typescript
export class AppError extends Error {
  constructor(
    public readonly code: string,
    message: string,
    public readonly statusCode: number = 500,
    public readonly details?: Record<string, unknown>,
  ) {
    super(message);
    this.name = 'AppError';
    // Maintain proper stack trace in V8
    if (Error.captureStackTrace) {
      Error.captureStackTrace(this, AppError);
    }
  }

  toJSON() {
    return {
      error: {
        code: this.code,
        message: this.message,
        ...(this.details ? { details: this.details } : {}),
      },
    };
  }
}
```

```typescript
import { createId as cuid2CreateId } from '@paralleldrive/cuid2';

/**
 * Generates a collision-resistant, URL-safe, monotonically sortable ID.
 * Uses @paralleldrive/cuid2 — NOT the deprecated `cuid` package.
 *
 * Use this for ALL entity IDs before Prisma insert operations.
 * The Prisma schema does NOT use @default() for IDs — IDs are app-generated.
 */
export function createId(): string {
  return cuid2CreateId();
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-PLAN04-01 | Information Disclosure | Stack traces in error responses | mitigate | `AppError.toJSON()` only returns `{ code, message, details }` — never `stack`. Fastify error handler (Plan 06) catches non-AppError and returns generic 500. |
| T-1-PLAN04-02 | Tampering | Email normalization bypass | mitigate | `SignupSchema` and `LoginSchema` both call `.toLowerCase()` on email — ensures consistent lookup regardless of case input |
| T-1-PLAN04-03 | Denial of Service | Password field length unbounded | mitigate | `LoginSchema` enforces `.max(100)` on password field; prevents bcrypt DoS attack from very long passwords (bcrypt silently truncates at 72 bytes — attacker could craft identical hashes) |
</threat_model>
