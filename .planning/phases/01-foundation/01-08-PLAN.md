**`apps/web/vite.config.ts`**:
```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';
import path from 'node:path';

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),  // Tailwind v4: Vite plugin, NOT PostCSS
  ],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,

  },
});
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-PLAN08-01 | Information Disclosure | Source maps in production build | mitigate | Vite's `build.sourcemap` defaults to `false` in production. Do not enable sourcemaps in prod build — they expose internal file paths and source code. |
| T-1-PLAN08-02 | Tampering | XSS via React innerHTML | accept | React escapes JSX by default. No `dangerouslySetInnerHTML` in Phase 1. Future plans using user-generated content must sanitize with DOMPurify. |
| T-1-PLAN08-03 | Information Disclosure | API URL exposed in frontend bundle | accept | API URL is `localhost:3000` in dev; production URL is not a secret (CORS protects endpoints). No API keys in frontend code. |
</threat_model>
